v1.0.1

  • 发布日期：2025-04-27

新增功能

平台架构升级：从 OpenClaw Manager 到通用 Agent 管理平台

  • 平台从仅支持 OpenClaw 的管理工具升级为通用 Agent 管理平台，新增 Agent 类型（agent_types）与渠道模板按类型关联的配置体系，为后续接入 Hermes、QwenPaw 等多种 Agent 打下基础。
  • 新增 SandboxSet（沙箱集合）CRD 管理能力，提供可视化的 SandboxSet 创建、编辑、删除界面，并在管理员初始化时自动创建内置 SandboxSet。
  • Hermes Agent 类型纳入平台管理，提供 Docker 配置与服务管理文件。

多语言国际化（i18n）

  • 全平台支持中英文双语切换，覆盖管理员控制台、用户控制台、登录注册、模型与渠道配置、SandboxSet/版本面板/Skills 引导等关键页面。

LiteLLM 网关支持

  • 模型渠道新增 LiteLLM 网关类型，支持与现有 OpenAI 兼容渠道并存。
  • 新增预算（Budget）配置：支持按用户/Token 设置额度上限，用户限额可同步到 LiteLLM 网关。
  • 模型与网关配置统一加密处理，配置字段拆分更清晰。

邮箱认证与密码管理

  • 新增邮箱注册 / 邮箱登录能力，并提供完整的密码重置流程。
  • 用户登录态优化，避免初始加载时重复拉取用户资料。

管理员侧能力增强

  • 管理员可以直接为用户创建实例，创建时若用户不存在会自动创建。
  • 实例操作记录审计、回滚机制完善。

ROS 升级触发数据库自动迁移

  • 引入 DbMigrateJob 机制：服务实例升级时由 ROS 自动执行 migrations/versions/<semver>/*.sql 增量迁移脚本，全程无需人工运维。
  • 内置基于 schema_migrations 表的幂等校验与 SHA256 内容校验，防止脚本被篡改导致的不一致。
  • 数据库初始化脚本支持复跑，升级场景可安全复用。

E2B 沙箱接入优化

  • 启用动态 DNS 解析 E2B 域名，自动判断走公网或私网，替代之前静态的 E2B_IP 配置。
  • 沙箱写入用户配置改为以 node 用户身份写入 /home/node，权限更安全。
  • 新增 E2B API 主机解析回退机制，提升不同网络环境下的可用性。

实例与运行时体验

  • 实例列表与详情展示真实 instance ID（而非 sandbox ID），便于用户对账与定位。
  • 新增「启动中」「停止中」状态展示，并补充实例恢复时的健康检查机制。
  • Supabase 改为优先通过内网（VPC）私有连接访问，降低跨网延迟与安全风险。

工程与文档

  • 新增 ESLint 严格模式 + pre-commit 检查，并补充新人开发指南。
  • 新增项目文档索引页面，便于查阅平台文档。

升级步骤

请参考 Release Notes 通用升级方式：在计算巢控制台对服务实例执行升级操作即可，数据库迁移会随升级流程自动完成。

注意事项

  • 本版本会执行数据库结构变更（OpenClaw 相关表迁移到通用 Agent 表结构，如 openclaw_instances → agent_instances），升级过程由 ROS DbMigrateJob 自动完成，请确保升级过程不要中断 ROS 资源栈。
  • 若此前自定义过 .env 中的 LiteLLM、邮箱服务相关变量，请参考最新的 .env.example 进行补全。
