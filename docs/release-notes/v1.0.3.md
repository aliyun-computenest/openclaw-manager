# v1.0.3

- **发布日期**：2025-05-30

## 版本概述
镜像： compute-nest-registry.cn-hangzhou.cr.aliyuncs.com/computenest/agent-manager:20260529_release_final_1

v1.0.3 在 v1.0.2 基础上聚焦于**可观测性、运维易用性、Agent 能力扩展与安全合规**，主要带来以下能力提升：

- **可观测性自动集成**：平台部署完成后自动接入 SLS 结构化日志收集，提供分级日志（业务、平台、运行时）的统一查询体验，无需用户手动配置 Logtail。
- **沙箱终端登录用户**：用户在控制台进入沙箱终端时使用专属业务账号登录，权限隔离更清晰。
- **Hermes 飞书渠道支持**：Hermes Agent 类型新增飞书（Feishu）渠道，可通过 App ID / App Secret 一键接入飞书机器人。
- **OpenClaw / Hermes Agent 镜像版本升级**：随发布更新到最新稳定版镜像，修复多项已知问题并增强功能。
- **Agent 配置易用性提升**：
  - 支持通过 `.env` 文件统一管理 Agent 环境变量；
  - AI 网关支持自动配置，部署即开箱可用；
  - 复制创建 Agent 类型时自动补全所有配置项，配置模板格式切换体验优化；
  - Skill 挂载文案与默认挂载路径优化。
- **安全合规加固**：
  - 新增升级前置确认参数，避免误操作；
  - 通过 RLS 限制非管理员用户自我提权，强化数据库行级权限。
- **国际化与体验细节**：沙箱升级界面国际化补全，请求 ID 在 API 日志中全链路传递，问题定位更高效。

## 升级步骤

1. [必须]请参考 [Release Notes 通用升级方式](./README.md#通用升级方式)：在**计算巢控制台**对服务实例执行**升级**操作，数据库增量迁移会随升级流程自动完成。
2. [可选]在Agent Manager 平台，**沙箱配置**中找到SandboxSet “agent-manager-openclaw” 和“agent-manager-hermes”，修改内容如下
   - 更新agent-manager-openclaw 和 agent-manager-hermes 中的镜像，分别为以下两个镜像
   
   `compute-nest-registry.cn-hangzhou.cr.aliyuncs.com/computenest/openclaw-manager-openclaw-test:2026.5.19`
   
   `compute-nest-registry.cn-hangzhou.cr.aliyuncs.com/computenest/openclaw-manager-hermes-test:2026.5.16`


> **重要**：v1.0.2 升级到 v1.0.3 时，由于 OpenClaw / Hermes Agent 镜像版本变更涉及配置兼容性调整（包括 Hermes 飞书渠道模板、startup_command、OpenClaw 钉钉模板等），需要在计算巢升级完成后**手动执行一次 Agent 配置更新**，详见 [Agent 版本升级指南](./v1.0.3_agent-version-upgrade-guide.md)。新部署的环境无需执行该步骤，初始化脚本已包含全部配置。

## 注意事项

- Hermes 飞书渠道依赖容器镜像预装的 `lark-oapi` 等 SDK，请确保使用本版本随附的 Hermes 镜像。
- SLS 日志收集功能默认启用，若已自建日志方案可在部署参数中关闭。
