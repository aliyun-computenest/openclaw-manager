# OpenClaw Agent 和 Hermes Agent 版本升级指南

> 适用场景：从 v1.0.2 升级到 v1.0.3 的**已有环境**。新安装环境无需执行，`init_database.sql` 已包含全部配置。

## 背景

v1.0.3 为 OpenClaw Agent 和 Hermes Agent 类型更新了镜像版本， 由于版本兼容问题，需要修改Agent的一些配置，并且为Hermes Agent添加了飞书渠道。由于以下 SQL 变更可能覆盖用户对Agent 配置的自定义修改，因此不通过自动迁移执行，改为手动操作。

## 变更内容概览

| 变更项 | 说明 | 原因 |
|--------|------|------|
| Hermes Agent 新增飞书渠道模板 | `channel_templates` 表插入 Hermes 的 feishu 记录 | 增加飞书渠道需要|
| Hermes Agent更新 startup_command | `.env` 中新增飞书相关环境变量 |增加飞书渠道需要 |
| Hermes Agent更新 modify_channel_command | 改用 `run-cmd.sh modify-channel` 统一处理渠道凭证 |增加飞书渠道需要 |
| OpenClaw Agent 钉钉模板更新 | 移除已废弃的 `gatewayPassword` / `gatewayToken` 字段 | 兼容Openclaw 新版本|
| OpenClaw Agent config_template 增强 | 添加 `gateway.http` 和 `plugins.entries` 配置 | 兼容Openclaw 新版本 |

## 操作方式说明

以下每个步骤均提供**两种等效方式**，任选其一即可：

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| **方式 A：平台界面手动修改** | 通过 Agent Manager 管理平台 UI 直接编辑对应配置 | 熟悉平台操作、变更量较少时推荐 |
| **方式 B：直接执行 SQL** | 在 Supabase SQL Editor 或 `psql` 中执行下方 SQL 语句 | 批量操作、脚本化部署时推荐 |

> **注意**：两种方式效果完全一致，请勿重复执行。

## 操作步骤

### 步骤 1：新增 Hermes 飞书渠道模板

#### 方式 A：平台界面修改

在 Agent Manager 平台进入 **Agent 类型管理 → Hermes → 渠道模板** 页面，新增一条飞书渠道模板，配置字段参考下方 SQL 中的 `config_fields` 内容。

#### 方式 B：执行 SQL

```sql
-- 为 Hermes 类型新增飞书渠道模板
INSERT INTO channel_templates (channel_type, name, description, config_fields, config_file, config_template, is_enabled, agent_type_id) VALUES
  ('feishu', '飞书 (Feishu)', '飞书开放平台应用配置',
    '[{"name": "clientId", "type": "text", "label": "App ID", "required": true, "placeholder": "cli_xxxxxxxxxxxxxxxx"}, {"name": "clientSecret", "type": "password", "label": "App Secret", "required": true, "placeholder": "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"}]',
    NULL,
    '{"_format": "yaml", "_content": ""}'::jsonb,
    true, (SELECT id FROM agent_types WHERE code = 'hermes'))
ON CONFLICT (channel_type, agent_type_id) DO UPDATE SET
    config_template = EXCLUDED.config_template,
    config_fields = EXCLUDED.config_fields;
```

### 步骤 2：更新 Hermes startup_command

> **注意**：此操作会覆盖 `agent_types` 表中 Hermes 的 `startup_command` 字段。如果您已手动修改过此字段，请先备份当前值，再根据下方模板合并您的自定义内容。

#### 方式 A：平台界面修改

在 Agent Manager 平台进入 **Agent 类型管理 → Hermes → 基础配置**，编辑 `startup_command` 字段，在 `.env` 写入部分新增飞书相关环境变量（参考下方 SQL 中的完整内容）。

#### 方式 B：执行 SQL

```sql
-- 查看当前值（可选，用于备份）
SELECT startup_command FROM agent_types WHERE code = 'hermes';

-- 更新 startup_command，增加飞书环境变量
UPDATE agent_types
SET startup_command = '#!/bin/bash

cat > /opt/data/.env << ''EOF''
DASHSCOPE_API_KEY=${DASHSCOPE_API_KEY}
DINGTALK_CLIENT_ID=${CHANNEL_CLIENT_ID}
DINGTALK_CLIENT_SECRET=${CHANNEL_CLIENT_SECRET}
GATEWAY_ALLOW_ALL_USERS=true
FEISHU_CONNECTION_MODE=websocket
FEISHU_APP_ID=${CHANNEL_CLIENT_ID}
FEISHU_APP_SECRET=${CHANNEL_CLIENT_SECRET}
FEISHU_DOMAIN=feishu
EOF

# 重启hermes
supervisorctl restart hermes'
WHERE code = 'hermes';
```

**新增的环境变量说明**：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `FEISHU_CONNECTION_MODE` | `websocket` | 飞书连接模式（固定值） |
| `FEISHU_APP_ID` | `${CHANNEL_CLIENT_ID}` | 飞书应用 App ID，来自平台占位符 |
| `FEISHU_APP_SECRET` | `${CHANNEL_CLIENT_SECRET}` | 飞书应用 App Secret，来自平台占位符 |
| `FEISHU_DOMAIN` | `feishu` | 飞书域名标识（固定值） |

### 步骤 3：更新 Hermes modify_channel_command

> **注意**：同样可能覆盖自定义修改，请先备份。

#### 方式 A：平台界面修改

在 Agent Manager 平台进入 **Agent 类型管理 → Hermes → 基础配置**，编辑 `modify_channel_command` 字段，改为通过 `run-cmd.sh modify-channel` 统一处理渠道凭证（参考下方 SQL 中的完整内容）。

#### 方式 B：执行 SQL

```sql
-- 查看当前值（可选，用于备份）
SELECT modify_channel_command FROM agent_types WHERE code = 'hermes';

-- 更新为通过 run-cmd.sh 统一处理渠道凭证
UPDATE agent_types
SET modify_channel_command = 'bash /usr/local/bin/run-cmd.sh modify-channel "${CHANNEL_TYPE}" "${CHANNEL_CLIENT_ID}" "${CHANNEL_CLIENT_SECRET}"
supervisorctl restart hermes'
WHERE code = 'hermes';
```

### 步骤 4：更新 OpenClaw 钉钉渠道模板

如果同时升级 OpenClaw 到 v5.18+，需要移除已废弃的 `gatewayPassword` / `gatewayToken` 字段。

#### 方式 A：平台界面修改

在 Agent Manager 平台进入 **Agent 类型管理 → OpenClaw → 渠道模板 → 钉钉**，编辑 `config_template`，移除 `gatewayPassword` 和 `gatewayToken` 字段（参考下方 SQL 中的目标值）。

#### 方式 B：执行 SQL

```sql
-- 移除 gatewayPassword / gatewayToken
UPDATE channel_templates
SET
  config_template = '{"dingtalk-connector": {"ackText": "任务已接收", "enabled": true, "clientId": "${CHANNEL_CLIENT_ID}", "clientSecret": "${CHANNEL_CLIENT_SECRET}", "groupSessionScope": "group", "separateSessionByConversation": true, "sharedMemoryAcrossConversations": false}}'::jsonb
WHERE channel_type = 'dingtalk'
  AND agent_type_id = (SELECT id FROM agent_types WHERE code = 'openclaw');
```

### 步骤 5：更新 OpenClaw config_template

如果需要启用 HTTP 端点和插件体系。

#### 方式 A：平台界面修改

在 Agent Manager 平台进入 **Agent 类型管理 → OpenClaw → 基础配置**，编辑 `config_template`，添加 `gateway.http` 和 `plugins.entries` 配置（参考下方 SQL 中的目标值）。

#### 方式 B：执行 SQL

```sql
-- 添加 gateway.http 和 plugins.entries
UPDATE agent_types
SET config_template = jsonb_set(
  jsonb_set(
    config_template,
    '{gateway,http}',
    '{"endpoints": {"chatCompletions": {"enabled": true}}}'::jsonb
  ),
  '{plugins}',
  '{"entries": {"openclaw-lark": {"enabled": true}, "dingtalk-connector": {"enabled": true}, "wecom-openclaw-plugin": {"enabled": true}, "openclaw-qqbot": {"enabled": true}}}'::jsonb
)
WHERE code = 'openclaw';
```
### 步骤 6：更新 OpenClaw 的SandboxSet

在SandboxSet的环境变量中添加以下内容

```
- name: OPENCLAW_GATEWAY_TOKEN
  value: xxx
```


## 注意事项

- 两种操作方式（平台界面 / SQL）效果等价，选择其一执行即可，请勿重复操作。
- Hermes 容器镜像需升级到包含飞书 SDK 预装的版本（`nousresearch/hermes-agent:v0.14.0` + 预装 `lark-oapi`、`dingtalk-stream` 等依赖），否则飞书渠道无法连接，如果您的环境中 `run-cmd.sh` 尚未包含飞书支持，需同步更新 Hermes 容器镜像。
