# 管理员侧 API 创建用户 —— 回归验证文档

> 验证目标：管理员通过 `POST /api/users` 直接创建用户，**当 email 在系统中不存在时，自动建好 Supabase auth user + `user_profiles` 档案**。

## 一、被验证的接口

| 接口 | 用途 | 鉴权 |
|---|---|---|
| `POST /api/users` | 单个创建用户（不存在则自动创建） | `requireAdmin` |
| `GET  /api/users?search=<email>` | 列表查询（用于回归校验） | `requireAdmin` |
| `DELETE /api/users/:userId` | 删除用户（用于测试清理） | `requireAdmin` |

源码位置：`openclaw-platform/server/routes/users.js`

### `POST /api/users` 请求体

```jsonc
{
  "email": "newuser@example.com",
  "username": "NewUser",
  "password": "at-least-6-chars",
  "role": "user",            // 可选，默认 user，可选 admin
  "maxInstances": 5,         // 可选，默认 5
  "authProvider": "email"    // 可选，默认 email；oauth/saml 时 password 可省略
}
```

成功响应（HTTP 200）：

```json
{
  "success": true,
  "user": {
    "id": "<uuid>",
    "email": "newuser@example.com",
    "username": "NewUser",
    "role": "user"
  }
}
```

失败场景（HTTP 4xx/5xx），常见 `error`：
- `email and username are required`
- `password is required for email authentication` / `password must be at least 6 characters`
- `User already registered`（email 已存在）

## 二、鉴权说明

后端 `requireAdmin` 校验的是 **Supabase 颁发的 JWT**，且要求该用户在 `user_profiles` 中 `role='admin'`。

获取 token 的标准流程：

```
POST  {VITE_SUPABASE_URL}/auth/v1/token?grant_type=password
Header: apikey: <VITE_SUPABASE_ANON_KEY>
        Content-Type: application/json
Body:   { "email": "<admin-email>", "password": "<admin-pwd>" }
Resp:   { "access_token": "<jwt>", ... }
```

`VITE_SUPABASE_URL` 和 `VITE_SUPABASE_ANON_KEY` 都可以直接 GET `<平台 URL>/env-config.js` 自动获取，无需手动配置。

## 三、本地回归运行

### 前置条件

- Python ≥ 3.8（仅用标准库，无任何第三方依赖）
- 你能从本机访问目标平台 `<PLATFORM_URL>`（形如 `http://<host>:<port>`）
- 该环境存在管理员账号且 `role='admin'`（默认 `admin@openclaw.local / admin123`）

### 运行

```bash
cd openclaw-platform/scripts

# 方式 1：通过参数传入平台地址
python3 verify-admin-create-user.py --platform-url http://<host>:<port>

# 方式 2：通过环境变量
export PLATFORM_URL=http://<host>:<port>
python3 verify-admin-create-user.py
```

完整可选参数：

```bash
python3 verify-admin-create-user.py \
  --platform-url http://<host>:<port> \
  --admin-email admin@openclaw.local \
  --admin-password admin123 \
  [--keep]                     # 保留测试用户，不清理
```

也可改用环境变量：

| 变量 | 是否必填 | 默认值 |
|---|---|---|
| `PLATFORM_URL` | **是**（或用 `--platform-url`） | _无_ |
| `ADMIN_EMAIL` | 否 | `admin@openclaw.local` |
| `ADMIN_PASSWORD` | 否 | `admin123` |
| `VITE_SUPABASE_URL` | 否 | _（默认从 `<PLATFORM_URL>/env-config.js` 抓取）_ |
| `VITE_SUPABASE_ANON_KEY` | 否 | _（同上）_ |

### 退出码

| 码 | 含义 |
|---|---|
| 0 | PASS |
| 1 | 断言失败（如：用户没被创建、重复创建未被拒绝等） |
| 2 | 脚本异常 / 网络错误 / 配置缺失 |

## 四、脚本执行的回归步骤

脚本每次跑都会用一个**带时间戳的全新 email**（如 `e2e-py-1761283200@openclaw.local`），避免污染。

1. **抓取 Supabase 配置** —— GET `${PLATFORM_URL}/env-config.js`，正则解析出 `VITE_SUPABASE_URL` 与 `VITE_SUPABASE_ANON_KEY`
2. **管理员登录** —— `POST {SUPABASE_URL}/auth/v1/token?grant_type=password` 拿到 `access_token`
3. **前置检查** —— `GET /api/users?search=<target_email>`，**断言目标 email 不存在**
4. **执行创建** —— `POST /api/users`，断言 `status=200 && body.success==true && body.user.id` 存在
5. **回归校验**
   - 5a. `GET /api/users?search=<target_email>`，断言能查到，且 `id` 与第 4 步返回一致；同时打印 `role / status / max_agent_instances` 等关键字段
   - 5b. **幂等性测试** —— 再次用同一个 email 调 `POST /api/users`，断言返回 4xx，证明重复创建被拒绝
6. **清理** —— `DELETE /api/users/<id>`（除非传了 `--keep`）

## 六、常见问题

- **`Admin login failed (status=400, body=invalid_grant)`**
  → 管理员账号或密码错误；或目标环境未初始化默认管理员。

- **`GET .../env-config.js returned HTTP 404`**
  → 后端没有 mount 静态前端（`dist` 不存在），或平台 URL 不对。可改用环境变量 `VITE_SUPABASE_URL` / `VITE_SUPABASE_ANON_KEY` 手动注入。

- **`POST /api/users` 返回 `403 Admin access required`**
  → 登录的账号在 `user_profiles` 里 `role` 不是 `admin`。

- **重复跑脚本不会冲突**
  → 每次 email 带时间戳；末尾会自动 `DELETE` 清理。如需保留可加 `--keep`。
